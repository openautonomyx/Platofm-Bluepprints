# Security Policy

## Supported versions

This repository template follows rolling updates.

## Reporting a vulnerability

- Do not disclose vulnerabilities publicly before coordinated remediation.
- Open a private security advisory or contact repository maintainers.
- Include reproduction steps, impact, and affected components.

## Security baseline

- Least privilege access
- Dependency and secret scanning
- SBOM and signed artifacts
- Tenant isolation and audit logging
- Secure defaults for cloud-native deployments
